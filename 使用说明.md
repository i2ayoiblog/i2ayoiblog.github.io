# 博客使用说明

这份文档覆盖从零开始到日常写作的完整流程，建议第一次按顺序操作，之后日常只需看「写文章」部分。

---

## 第一步：安装 Hugo

Hugo 是驱动博客的引擎，安装只需下载一个文件。

1. 打开浏览器，访问：https://github.com/gohugoio/hugo/releases/latest
2. 在页面找到 `hugo_extended_0.xxx.x_windows-amd64.zip`，点击下载（选 `extended` 版本）
3. 解压后得到 `hugo.exe`
4. 将 `hugo.exe` 放到 `C:\Windows\System32\` 文件夹（这样任何位置都能用）
5. 打开「命令提示符」或「PowerShell」，输入以下命令验证安装成功：

```
hugo version
```

看到版本号输出就说明安装成功了。

---

## 第二步：注册 GitHub 账号

GitHub 用来托管你的博客代码，网站发布后通过它自动更新。

1. 打开 https://github.com/signup
2. 填写邮箱、密码、用户名（用户名会出现在博客地址里，比如用户名是 `john`，博客地址就是 `john.github.io`，建议选一个简短好记的）
3. 完成邮箱验证
4. 登录后，点击右上角头像 → Settings → 确认账号信息

---

## 第三步：初始化 Git 并推送代码

完成前两步后，在博客目录（D:\BLOG）里依次执行以下命令。

**先配置 Git 身份（只需做一次）：**

```
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
```

**初始化仓库并推送：**

```
cd D:\BLOG
git init
git add .
git commit -m "初始化博客"
```

**在 GitHub 创建新仓库：**

1. 打开 https://github.com/new
2. Repository name 填写：`你的用户名.github.io`（例如 `john.github.io`）
3. 选 Public（公开）
4. **不要**勾选任何初始化选项（README、.gitignore 都不勾）
5. 点击 Create repository

**推送代码到 GitHub（把下方的 `你的用户名` 替换成真实用户名）：**

```
git remote add origin https://github.com/你的用户名/你的用户名.github.io.git
git branch -M main
git push -u origin main
```

---

## 第四步：开启 GitHub Pages 自动部署

1. 打开你的仓库页面：`https://github.com/你的用户名/你的用户名.github.io`
2. 点击顶部菜单的 **Settings**
3. 左侧栏点击 **Pages**
4. 在 Source 下拉菜单选择 **GitHub Actions**
5. 保存

完成后，每次你 `git push`，GitHub 会自动用 Actions 构建并发布网站，通常 1~2 分钟完成。

**第一次部署后，访问 `https://你的用户名.github.io` 就能看到博客了。**

---

## 第五步：修改博客名称

目前博客名称是占位的「我的博客」，想改的时候打开：

```
D:\BLOG\hugo.toml
```

修改这两行：

```toml
title = "你想要的博客名"
```

```toml
subtitle = "你的副标题"
```

改完后保存，然后执行：

```
cd D:\BLOG
git add hugo.toml
git commit -m "修改博客名称"
git push
```

## 日常操作：写一篇新文章

这是你以后最常用的流程。

### 1. 新建文章文件

在对应分类的文件夹里新建一个 `.md` 文件：

- 酒 → `D:\BLOG\content\wine\文件名.md`
- 学习笔记 → `D:\BLOG\content\study\文件名.md`
- 游戏 → `D:\BLOG\content\gaming\文件名.md`

文件名建议用英文或拼音加连字符，比如 `yamazaki-12.md`，不影响页面显示的标题。

### 2. 文章模板

每篇文章开头的格式是固定的（叫 Front Matter），把下面内容复制进去改一改：

```markdown
---
title: "文章标题"
date: 2026-06-25
draft: false
categories: ["酒"]
tags: ["威士忌", "日本威士忌"]
description: "这里写一句话简介，会显示在文章列表里"
---

正文从这里开始，支持 Markdown 语法。
```

**draft: false** 表示正式发布；如果想先存草稿不发布，改成 `true`。

### 3. 本地预览（可选但推荐）

在发布之前可以先在本地看效果：

```
cd D:\BLOG
hugo server
```

然后打开浏览器访问 `http://localhost:1313`，实时预览，保存文件后页面自动刷新。

按 `Ctrl + C` 停止预览服务器。

### 4. 发布文章

确认没问题后，执行：

```
cd D:\BLOG
git add .
git commit -m "新增文章：文章标题"
git push
```

1~2 分钟后博客自动更新。

## 新增分类

目前有三个分类：酒、学习笔记、游戏。想加新分类，比如「旅行」：

**第一步**，在 `content` 下建文件夹：

```
D:\BLOG\content\travel\
```

**第二步**，打开 `hugo.toml`，在 `[menu]` 部分加一条：

```toml
[[menu.main]]
  name = "旅行"
  url = "/categories/旅行/"
  weight = 5
```

`weight` 数字越小越靠前，按顺序排列就好。

**第三步**，写文章时 `categories: ["旅行"]` 就会自动归入新分类。

## 常见问题

**Q：推送后网站没更新？**  
A：打开 `https://github.com/你的用户名/你的用户名.github.io/actions`，看 Actions 有没有报错。

**Q：本地预览没问题，线上样式乱了？**  
A：打开 `hugo.toml`，确认 `baseURL` 已经改成你的真实地址，比如 `https://john.github.io/`。

**Q：想删除一篇文章？**  
A：直接删除对应的 `.md` 文件，然后 `git add . → git commit → git push` 即可。

**Q：想修改已发布的文章？**  
A：直接编辑对应的 `.md` 文件，保存后 push 即可，历史不会丢失。

**Q：图片怎么添加？**  
A：把图片放到 `D:\BLOG\static\images\` 里，比如 `photo.jpg`，然后在文章里用：
```markdown
![图片描述](/images/photo.jpg)
```

## 目录结构速查

```
D:\BLOG\
├── hugo.toml              ← 博客配置（名称、菜单、参数）
├── content\               ← 所有文章都在这里
│   ├── wine\              ← 酒类文章
│   ├── study\             ← 学习笔记
│   ├── gaming\            ← 游戏文章
│   └── about.md           ← 关于页
├── static\
│   └── images\            ← 放图片的地方
├── themes\simple\         ← 博客主题（不需要动）
└── .github\workflows\     ← 自动部署配置（不需要动）
```
